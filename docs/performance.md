# 性能試験記録

## 実行方法

Python 3.12環境で次を実行する。スクリプトはOSの一時領域だけを使い、利用者データを変更しない。

```console
.venv/bin/python tools/measure_performance.py --count 10000 --runs 20
```

## 初回計測（2026-07-16）

- 環境: Linux、Python 3.12.3、SQLite（Python標準ライブラリ同梱版）
- データ: 案件1件、対象20件、脆弱性メモ10,000件
- 一覧 p95: 0.000176秒
- 本文部分一致検索 p95: 0.001797秒
- 複合絞り込み p95: 0.000034秒
- 判定: DBクエリはいずれも2秒目標以内

一覧は `idx_notes_updated_at`、複合絞り込みは複合インデックスを利用する。先頭ワイルドカードを許可する本文部分一致検索は全件走査になるが、1万件の初回計測では目標以内だった。データ分布や端末差があるため、リリース前に同じスクリプトで再計測する。

この値はSQLiteクエリだけの計測であり、ブラウザ描画やウイルス対策ソフト等の時間は含まない。通常操作全体の95%目標は、対象OS・ブラウザの実機試験と併せて最終判定する。
